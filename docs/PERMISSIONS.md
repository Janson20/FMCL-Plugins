# FMCL 权限系统参考

> 本文档列出 FMCL 插件系统支持的所有权限、风险等级及使用场景。

---

## 权限总览

| 权限标识 | 显示名称 | 风险等级 | 管理方式 |
|----------|----------|----------|----------|
| `filesystem.read` | 读取文件 | 低 | 安装时一次性确认 |
| `network.http` | HTTP 请求 | 低 | 安装时一次性确认 |
| `ui.extend` | 注册 UI 组件 | 低 | 安装时一次性确认 |
| `ui.notification` | 发送通知 | 低 | 安装时一次性确认 |
| `data.store` | 数据存储 | 低 | 安装时一次性确认 |
| `filesystem.write` | 写入文件 | 中 | 安装时确认 + 可撤销 |
| `core.download` | 自定义下载源 | 中 | 安装时确认 + 可撤销 |
| `core.version` | 自定义版本源 | 中 | 安装时确认 + 可撤销 |
| `data.settings` | 修改设置 | 中 | 安装时确认 + 可撤销 |
| `network.socket` | 原始 Socket | 高 | 每次触发时弹窗确认 |
| `core.launch_hook` | 启动钩子 | 高 | 每次触发时弹窗确认 |
| `core.process` | 执行外部进程 | 高 | 每次触发时弹窗确认 |

## 各权限详细说明

### `filesystem.read` - 读取文件

允许插件读取文件系统中的文件。

- **允许范围**: 启动器核心目录 + `.minecraft` 目录 + 插件数据目录
- **禁止范围**: 系统文件、其他用户目录
- **典型用途**: 读取配置文件、日志文件、模组元数据

### `filesystem.write` - 写入文件

允许插件写入/修改文件系统中的文件。

- **允许范围**: `.minecraft` 目录 + 插件数据目录
- **禁止范围**: 启动器核心目录、系统关键目录
- **典型用途**: 修改游戏配置文件、保存模组数据、备份存档

### `network.http` - HTTP 请求

允许插件发送 HTTP/HTTPS 网络请求。

- 必须使用启动器的 User-Agent 代理
- 默认 15 秒超时
- **典型用途**: 调用 REST API、下载文件、数据上报

### `network.socket` - 原始 Socket

允许插件建立原始 TCP/UDP Socket 连接。

- **高风险**: 可用于端口扫描、代理转发等危险操作
- 每次连接前需用户确认
- **典型用途**: 联机助手、服务器状态检测

### `ui.extend` - 注册 UI 组件

允许插件在主界面注册标签页、侧边栏项目、设置面板。

- UI 组件由插件自行实现（需遵循 CustomTkinter/Tkinter 规范）
- **典型用途**: 自定义标签页、工具面板、数据可视化

### `ui.notification` - 发送通知

允许插件向用户发送 Toast 弹窗通知。

- 支持 info / success / warning / error 四个级别
- **典型用途**: 操作结果提示、状态变化通知

### `core.download` - 自定义下载源

允许插件向启动器注册额外的下载来源。

- 可注册自定义 URL 前缀映射
- **典型用途**: 第三方镜像源、私有资源服务器

### `core.version` - 自定义版本源

允许插件向启动器注册额外的 Minecraft 版本来源。

- 可添加非官方版本列表
- **典型用途**: 第三方整合包分发平台

### `core.launch_hook` - 启动钩子

允许插件在游戏启动前后执行自定义逻辑。

- 可修改启动命令、JVM 参数、gameDirectory
- **高风险**: 误操作可导致游戏无法启动
- **典型用途**: 启动前备份、JVM 参数优化、启动后监控

### `core.process` - 执行外部进程

允许插件启动外部程序。

- **高风险**: 可能被用于执行恶意程序
- 每次执行前需用户确认
- **典型用途**: 启动辅助工具、调用命令行工具

### `data.store` - 数据存储

允许插件在 `plugins/data/{plugin_id}/` 目录中读写持久化数据。

- 每个插件独立目录，互不干扰
- **典型用途**: 统计数据存储、缓存文件、本地数据库

### `data.settings` - 修改设置

允许插件读取和修改启动器的全局设置。

- 可修改 config.json 中的部分字段
- **典型用途**: 根据插件逻辑自动调整启动器行为

## 权限管理流程

```
安装 .fmpl
  ├─ 解析 plugin.json 中的 permissions 字段
  ├─ 弹出权限确认弹窗（按风险等级分级展示）
  ├─ 用户确认 → 授权所有请求的权限
  └─ 用户拒绝 → 安装失败
```

```
启用已安装插件
  ├─ 检查是否有未授权的高风险权限
  ├─ 有 → 弹出权限确认弹窗
  ├─ 无 → 直接启用
  └─ 调用 PluginBase.on_enable()
```

```
运行时高风险权限使用
  ├─ 插件调用需要高风险权限的操作
  ├─ 检查「始终允许」标记
  ├─ 已标记 → 直接执行
  ├─ 未标记 → 弹出运行时确认弹窗
  └─ 用户选择：
       ├─ 允许本次 → 执行
       ├─ 始终允许 → 执行 + 设置标记
       └─ 拒绝 → 不执行
```

---

## 最佳实践

1. **最小权限**: 只申请插件实际需要的权限，不要申请过度
2. **诚实声明**: 在 `plugin.json` 中填写完整的 permissions 列表
3. **描述理由**: 在插件描述中说明为什么需要每个权限
4. **尊重用户**: 高风险权限被拒绝后，插件应有降级处理，优雅地跳过而非崩溃
5. **合规性**: 不使用权限进行未声明的操作
